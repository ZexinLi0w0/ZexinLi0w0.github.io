2.9

TX2 jetpack 4.6 + tensorflow r2.6/r2.4均能够编译且能够pip安装。但是python执行时，初始化变量就python进程卡住(此时无法被control + C结束进程，只能kill)。htop观察资源利用率发现一个CPU核心100%占用，GPU 0%占用。回退jetpack版本到4.5.1, 再编译tensorflow r2.4/r2.6仍然有此问题。计划回退jetpack版本到3.3再尝试编译。

2.3

在TX2上编译tensorflow r2.6，目前没报错。编译成功后出NVIDIA Jetson TX2编译tensorflow 2.6教程。

环境准备: 刷机jetpack 4.6, 其他参考tensorflow官方编译指南 NVIDIA编译tensorflow指南。 注意查询Jetpack和tensorflow版本的兼容性，可以发现jetpack 4.6支持tensorflow 2.5/2.6/2.7。虽然编译2.6虽然能编译+安装成功，但是会出现ARM64 NUMA return 0的报错并卡住python进程，原因如下，官方并没有修复这个问题并建议使用tensorflow 2.5。然而，对于r2.5而言，在TX2上会直接发生@local_cuda not found的error，无法编译原因不明（可能需要编译指定版本的CUDA）。

潜在问题

  1. pip依赖全部用root配置，否则h5py可能会编译不过

  2. 编译tensorflow时不能处于root账户, 也不要用sudo。否则会出现local_cuda_not_found的编译错误

  3. 一定要先设置8GB交换区，否则会出现c++ compiling error跟随出现process xx killed

  4. 需要至少32GB外置储存设备，NVIDIA Jetson TX2/AGX的内置储存不足以支持编译

  5. cannot import name ‘function_pb2’: 不要在tensorflow源代码repo import tensorflow
